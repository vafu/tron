# New stream pipeline validates V4L MJPG metadata loop

Timestamp: 2026-05-08T13:48:55-07:00

Question / hypothesis: Can the new stream architecture run as a minimal future-main binary with V4L source, optional decoder pipeline, and metadata logging before adding rendering?

Observed result: Added src/bin/tron-pipeline.rs using V4lFrameSource, DecodeStream/PassthroughStream, and TurboMjpegDecoder. Ran default Lenovo path /dev/video53, RGB, 1280x720 MJPG, 30 fps, 4 mmap buffers. The pipeline produced 30 fps steady-state, decoded to PixelFormat::Bgra8 with stride 5120 and len 3686400, and carried V4L sequence plus monotonic start-of-exposure timestamps.

Follow-up tasks / instrumentation gaps: Next steps are to add structured telemetry output, then add a render sink around FrameStream. Keep this binary small and use it as the new-main integration point instead of growing tron-stream.
