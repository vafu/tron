# Task design depth fusion baseline

Timestamp: 2026-05-02T13:22:25-07:00

Tags: #tron #task

Task:

Design the first `z` estimation baseline for the cube-drag demo.

Why:

The demo needs a stable enough `x`, `y`, `z` pointer to move a cube. It does not need perfect metric depth initially, but it needs predictable relative depth and confidence.

Candidate signals:

- RGB landmark center for `x` and `y`.
- Landmark bounding-box scale or palm size for relative `z`.
- IR foreground area/intensity as a correction signal when valid.
- Proximity sensor as a near-field confidence or clamp.
- Temporal filtering to suppress jitter.

Output:

A proposed `PointerState` mapping with confidence rules and failure modes, ready to implement behind a new pipeline stage.
