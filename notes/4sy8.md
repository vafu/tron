# Corrected signal behaves as transient depth cue

# Corrected signal behaves as transient depth cue

Timestamp: 2026-05-07T18:30:07-07:00

Tags: #tron

Question / hypothesis: Can the normalized IR diff `corrected_signal` provide a
useful depth cue despite camera auto-exposure?

Observed result: User observed that `corrected_signal` is extremely good for
fast hand motions: it spikes clearly with rapid approach/retreat. During slow
motions or held positions, the signal gradually moves back toward baseline as
camera exposure adapts. This suggests the current metric is a strong transient
depth/proximity cue but not an absolute steady-state depth estimate.

Follow-up tasks / instrumentation gaps: Fuse the transient IR cue with landmark
scale/position dynamics. Track ROI size, landmark span, corrected-signal delta,
and signal decay rate together so the estimator can preserve depth state while
auto-exposure relaxes the raw brightness signal.
